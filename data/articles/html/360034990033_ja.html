CircleCI上の権限

CircleCI 上のユーザ権限は、 GitHub や Bitbucket 上のユーザ権限に同期しています。 管理者権限を持つすべてのユーザーはCircleCI の管理者となります。 ページのロードごとにVCSのアクセス権限をチェックします。

GitHub

Github上 のオーナーであり、CircleCI でビルドされている1 つ以上のリポジトリをフォローしてるユーザは組織の管理者となります。

GitHub の権限については、このサイトで参照できます（https://help.github.com/en/articles/permission-levels-for-an-organization）

Bitbucket

Bitbucket上 の管理者であり、CircleCI でビルドされている 1 つ以上のリポジトリをフォローしているユーザは組織の管理者となります。

トラブルシューティング

CircleCI上で権限が更新されていないなどの問題が発生した場合は、次のステップを試してください。

  1. 他のブラウザや他のデバイスでシークレットウィンドウを使ってCircleCIにアクセスします。 ブラウザのキャッシュの問題である可能性があり、その場合、ブラウザでキャッシュをクリアする必要があります。 他の端末やシークレットウィンドウなどで問題が解消される場合は、*.circleci.com および circleci.com に関連するブラウザキャッシュをクリアしてください。それ以外の場合は、circleci.com/account ページの[Refresh Permissions] をクリックすることでユーザ権限を更新します。
  2. 上記の手順でも問題が解決されない場合、組織名、ユーザー名、VCS プロバイダー（GitHub または Bitbucket）を明記して、サポートチケットを提出いただくか、support@circleci.com にお問い合わせください。 CircleCI サポートが、CircleCI 側のキャッシュをクリアし、古くなった、または破損した可能性のある権限を更新します。
